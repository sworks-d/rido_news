# themes.md

## テーマ定義

ルート・スポット記事の切り口となるテーマ定義。
カテゴリで切るのではなく、ライダーの「行きたい理由」で切る。
Agent 2がデータに合わせて自動選択する。
参謀・整備士がパフォーマンスデータを元に定期更新する。

## グルメカテゴリの扱い
現フェーズでは優先度低。
道の駅タグとの組み合わせのみ使用する。
単体のグルメテーマはPh2以降に追加する。

---

## テーマ選択ロジック（Agent 2向け）

収集データのタグ・バイク種別・距離・seasonを照合して
最もマッチするテーマを自動選択する。

優先順位：
1. バイク種別が判定できる場合 → バイク種別系を優先
2. 季節が明確な場合 → 季節系を優先
3. カテゴリ横断が可能な場合 → カテゴリ横断系を優先
4. 上記に該当しない場合 → 道の性質系・単体深掘り系

## テーマレベルアップルール

### 計測期間
毎週月曜に参謀が集計する。

### テーマのステータス定義

| ステータス | 条件 | 配信頻度 |
|---|---|---|
| Lv1 新規 | 追加直後 | 週1回 |
| Lv2 成長 | CTR10%以上・2週間継続 | 週2回 |
| Lv3 人気 | CTR15%以上・4週間継続 | 週3回 |
| 休眠 | CTR5%未満・2週間継続 | 配信停止 |

### 休眠から復活する条件

#### ユーザー行動ベース
休眠テーマに関連するタグのルート・スポットが
直近7日間で10件以上投稿された場合

#### 季節ベース
季節系テーマは対応季節の1ヶ月前に自動復活

#### 手動復活
リーダーがDiscordから指示して即時復活

### レベルダウン
```
Lv3 → Lv2：CTR10%未満が1週間継続
Lv2 → Lv1：CTR5%未満が1週間継続
Lv1 → 休眠：CTR5%未満が2週間継続
```

### 新テーマ追加基準
以下のいずれかに該当する場合に参謀がリーダーに提案する。
```
1. 既存テーマでカバーできないタグの組み合わせが
   直近2週間で15件以上蓄積された場合
2. 季節イベントの1ヶ月前
3. リーダーが手動で追加指示した場合
```

### 知見の蓄積
全テーマの履歴をSupabaseのthemes_logに記録する。
休眠テーマも削除せず履歴を保持する。
```json
{
  "theme_id": "xxx",
  "theme_name": "川沿いと滝を巡る",
  "status": "sleeping",
  "level": 0,
  "ctr_history": [0.12, 0.08, 0.04, 0.03],
  "revived_count": 2,
  "last_revived_at": "2026-03-01",
  "created_at": "2026-01-01"
}
```
