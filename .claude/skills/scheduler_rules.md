| 月 | 東北 | 青森・岩手・宮城・秋田・山形・福島 |

## 2. 配信スケジュール

| 時間 | 内容 | 対象タブ |
|---|---|---|
| 毎朝6時 | バイクニュース最新化 | バイクニュース |
| 毎朝6時 | 当日エリアのルート・スポット配信 | ルート・スポット |
| 金曜18時 | 週末モード強化配信 | ルート・スポット |
| 月曜5時 | 週次集計リセット・新特集生成 | 全タブ |
| 随時 | 緊急ニュース（法律改正・リコール等） | バイクニュース |

---

## 3. 収集→配信パイプライン

### 通常配信（毎朝）
```
前日23時　収集パイロット実行
前日24時　生成パイロット実行
当日1時　 品質パイロット判定完了
当日2時　 配信キューに投入
当日6時　 表示切り替え
```

### 週次リセット（月曜）
```
日曜23時　収集パイロット集計開始
月曜1時　 タブ切り替え判定
月曜3時　 新特集生成完了
月曜5時　 全タブ更新
```

### 緊急配信
```
収集パイロットが緊急フラグを検知
        ↓
生成パイロットが即時生成（通常キューを飛ばす）
        ↓
品質パイロット判定（Layer 1のみ）
        ↓
Discordにアラート通知
        ↓
リーダー承認後即時配信
```

---

## 4. カテゴリバランスルール

```
同一カテゴリ → 1日2件まで
同一メーカー → 1日1件まで
緊急ニュース → バランスルール適用外
```

超過する場合は翌日のキューに回す。

---

## 5. データなし時の挙動

```
当日エリアのルート・スポットが0件
→ 全国人気スポットをフォールバック表示

バイクニュースが0件
→ 前日の記事を再表示（鮮度ラベルを更新）

フォールバック発生時は参謀にDiscord通知
```

---

## 6. コスト管理

```
1日あたりのAPI呼び出し上限：500回
月間コスト上限：設定値を超えたら参謀がアラート
緊急配信はコスト上限の対象外
```

---

## 7. 将来の更新について

曜日×エリアの割り当てはユーザー分布の変化に応じて見直す。
更新時は整備士がこのファイルを修正する。
変更はリーダーの承認後に反映する。
