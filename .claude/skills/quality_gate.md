# quality_gate.md

## 品質判定基準

生成された記事をSupabaseに投入する前に物理的に止める防衛システム。
AIにルールを書くだけでは必ず破る。hookで強制する。

---

## Layer 1：即リジェクト

1つでも該当したら問答無用で承認待ちキューへ。
Discordにアラート通知。

### 著作権リスク（external_rssのみ）
```
summary文字数 / body文字数 > 0.2 → リジェクト
bodyから15字以上の連続フレーズがsummaryに含まれる → リジェクト
タイトルと完全一致する文字列がsummaryに含まれる → リジェクト
```

### 有害表現
差別・暴力・誹謗中傷ワードの検出 → リジェクト

### 事実誤認リスク（external_rssのみ）
summaryに含まれる数値・日付・法律名をbodyと照合
不一致が1つでもある → リジェクト

### スポンサーバッティング
掲載中スポンサーの競合他社への批判的言及 → リジェクト
（批判的表現例：「〜より劣る」「〜はダメ」「〜を避けるべき」）

### 危険運転助長
速度無制限・信号無視・無免許・法定速度を超えて等の表現 → リジェクト

## Layer 2：品質スコア

100点満点で採点する。70点未満はアラート。

| 項目 | 配点 | 基準 |
|---|---|---|
| 文字数 | 20点 | カテゴリ別基準内か |
| 構成 | 20点 | 必須フィールドが揃っているか |
| 情報鮮度 | 20点 | ソース記事が7日以内か（app_dbは常に20点） |
| カテゴリ整合 | 20点 | categoryとsummaryの内容が一致するか |
| ソース信頼性 | 20点 | trust_score基準（app_dbは常に20点） |

### 文字数基準
```
bike_news: summary 100〜200字
route: summary 120〜200字・sections各200字以内
spot: summary 120〜200字・sections各200字以内
```

## Layer 3：トーンチェック

rido_tone.mdの禁止事項と照合する。

### 自動修正対象
```
命令調の変換
「〜してください」→「〜するといい」
「〜しましょう」→「〜するのもあり」
「〜すべきです」→「〜かもしれない」

感嘆符の削減
4個以上の感嘆符 → 2個に削減

締めの修正
「いかがでしたか？」→ 削除
「ぜひ参考にしてください」→ 削除
```

自動修正後に再度Layer 3を実行する。

### 即アラート対象
```
ランキング表現：「1位」「2位」「ランキング」「TOP〇」
根拠なき最上級：「日本一」「最強」「No.1」
煽り系：「驚きの」「衝撃の」「まさか」
```

## hookルール

DB投入前に以下をhookで物理チェックする。

```
hook_1: external_rssでsource_urlがnull → 投入拒否
hook_2: titleが30字超 → 投入拒否
hook_3: summaryが250字超 → 投入拒否
hook_4: content_type=routeでroute_idがnull → 投入拒否
hook_5: content_type=spotでspot_idがnull → 投入拒否
hook_6: 同一article_idが既にpublished → 投入拒否
hook_7: index数とsections数が不一致 → 投入拒否
```

## 判定結果の分岐

### 自動公開キュー
Layer 1〜3全て通過した記事。
Discordには日次サマリーで報告。

### 承認待ちキュー
Layer 1で引っかかった記事。
Discordに即時アラート。

### 自動修正後再判定
Layer 3で修正対応した記事。
修正後に再度Layer 2〜3を通す。

## リンクポリシー

### 引用元URLの表示
source_type = 'external_rss'の記事は引用元URLを必ず付与する。

### リンクの挙動
- 外部RSS記事 → アプリ内WebViewで開く
- アプリ内データ → ネイティブ遷移（route_id / spot_id）
- PR/スポンサー → 外部ブラウザで開く
- それ以外 → 外部ブラウザ起動禁止

### Agent 生成時のルール
- source_type = 'external_rss'の場合：source_urlフィールドを必ず含める
- source_type = 'app_db'の場合：route_idまたはspot_idを必ず含める

## エスカレーションルール

### 即時Discord通知
- Layer 1引っかかり
- Layer 3の即アラート項目検出

### 日次サマリーに含める
- Layer 2スコア70点未満
- 自動修正実施件数

### 整備士への改善依頼トリガー
- 同じLayer 1理由が3件連続
- 自動修正失敗率が20%超
- Layer 2平均スコアが75点未満・2週間継続

参謀に報告する。
